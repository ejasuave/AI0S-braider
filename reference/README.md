# Project Braids — Reference Library

Authoritative documents for AI-assisted development. **Read these before writing code.**

## Document hierarchy

| Priority | Document                                                         | Path                                                                         |
| -------- | ---------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| 1        | **Product Blueprint** — product, brand, stack, AI rules, roadmap | [requirements/product-blueprint.md](requirements/product-blueprint.md)       |
| 2        | **Engineering Playbook** — per-system architecture               | [requirements/engineering-playbook.md](requirements/engineering-playbook.md) |
| 3        | **Prompt Library** — build order and prompts                     | [prompt-library/](prompt-library/)                                           |
| 4        | **Design spec** — visual identity and UX                         | [design/visual-identity-and-ux.md](design/visual-identity-and-ux.md)         |
| 5        | **Pitch** — GTM context only                                     | [requirements/detailed-pitch.md](requirements/detailed-pitch.md)             |

**Conflict resolution:** Founder decision → Product Blueprint → Back Matter → Playbook → Pitch.

## Folder guide

| Folder            | Use for                                      |
| ----------------- | -------------------------------------------- |
| `requirements/`   | Blueprint, Playbook, pitch                   |
| `prompt-library/` | Outline, back matter, future chapter prompts |
| `design/`         | Mockups, UI notes, visual identity           |
| `api/`            | OpenAPI specs, endpoint examples (as built)  |
| `notes/`          | Scratch notes — not authoritative            |

## Supported formats

Text (`.md`, `.json`, `.yaml`), images (`.png`, `.jpg`, `.webp`), code samples.
